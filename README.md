# vedomosty

Внешняя обработка создана на базе 1С:Предприятие 8.3, учебная версия (8.3.27.1688).

Реализован функционал согласно ТЗ кнопок "Заполнить нужное" и "Удалить лишнее".
Для тестирования на стороне пользователя были заполнены справочник Договоры, документ Ведомость 
и регистр сведений История статусов документов - согласно таблицам из ТЗ.
*********************

Реализованы опции на форме:

***1.Выбор типа даты – равно или не равно.

***2.Дата равна минимальной или максимальной.

В ТЗ текст:
дата начала действия первого активного договора в табличной части ведомости равна 
(или не равна1) минимальной (или максимальной2) дате начала действия договора в этой табличной части.

***3.Дата отправки ТЗ.
Исключение ведомостей при заполнении таблицы.

***4.Выбор пола.

Создание условного оформления для выделения строк цветом.

Здесь чтобы отличалось, что сработал выбор женского пола – подсвечивается красным. 

Столкнулся с проблемой (победить не смог) – чтобы корректно работ данный функционал при 
переключении пола после того, как таблица построена, надо чтобы был реализован только 
один способ подсветки, или в настройках конфигуратора по условию, или в коде процедуры. 
Сейчас работает согласно ТЗ на стадии формирования таблицы. По умолчанию, без выбора 
пола подсветка производится в светло-зеленый.

***5.Фамилия кандидата. 

В зависимости от количества введенных знаков реализован разный 
тип сообщения пользователю.
